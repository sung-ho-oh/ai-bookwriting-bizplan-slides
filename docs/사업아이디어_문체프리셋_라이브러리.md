## 한 줄 개념

> 대표가 설계한 **몇 가지 완성형 문체 스타일(프리셋)**을 라이브러리로 제공하여,
> 저자가 ①마음에 드는 스타일을 고르거나 ②그 스타일을 기반(baseline)으로 자신의 문체를 발전시키도록 한다.

이것은 build-voice(백지에서 내 글로 문체를 뽑는 것)와 **상호 보완**이다.
- build-voice = 상향식(내 글 → 내 문체)
- 문체 프리셋 = 하향식(완성형 스타일 → 선택/변형 → 내 문체)
- 결합: 프리셋을 출발점으로 골라 두고, 내 글이 쌓이면 build-voice로 개인화 → 두 방향이 만난다

## 왜 대표의 지적 자산이 되는가

- 프리셋 자체가 **대표가 30년 HR·집필 경험으로 큐레이션한 저작물** — 복제 불가한 노하우
- 배포본에는 프리셋의 '적용 결과'만 나가고 **설계 원본(왜 이 스타일이 이렇게 구성되는가)은 대표만 보유** → IP 보호 자연 성립
- 코칭·강의·라이선스의 차별화 포인트 ("우리만의 문체 스타일 라이브러리")

## 프리셋 스타일 후보 (초안 — 확정은 대표)

| 스타일명(가안) | 성격 | 어울리는 책 |
|---|---|---|
| 신뢰의 실용서 | 하십시오체, 짧고 단단한 문장, 근거 우선 | 비즈니스·자기계발 |
| 따뜻한 멘토 | 해요체, 경험담·비유 풍부, 독자에게 말 걸기 | 에세이·인생론·회고 |
| 강연자의 목소리 | 구어체 리듬, 질문 던지기, 강조 반복 | 강의 기반서·동기부여 |
| 학자의 정갈함 | 정확한 개념어, 논리 전개, 절제된 톤 | 교양·전문 입문서 |
| 이야기꾼 | 장면 묘사, 서사적 도입, 감정선 | 스토리텔링·자전적 |

> 각 프리셋 = 6개 축(어미·문장리듬·어휘·독자말걸기·사례방식·금지) 값 + 대표 예문 세트로 정의.
> build-voice의 6축 구조를 그대로 재사용하므로 기술적 연속성 있음.

## 실행 스케치 (여유 있을 때)

1. **프리셋 정의 파일** 포맷 확정 (6축 값 + 예문 3편 + 설명). 대표가 5종 내용 집필, 내가 구조화.
2. **choose-voice 스킬**(가칭): "문체 스타일 골라줘" → 5종을 예문과 함께 보여주고 선택 → `_Project/문체가이드.md`에 그 프리셋을 기준선으로 세팅.
3. **혼합 모드**: "이 스타일 기반으로 내 글 반영해줘" → 선택 프리셋 + build-voice 분석을 가중 결합.
4. 온보딩 모듈 C 분기: 샘플 없는 저자 → 프리셋 선택 / 샘플 있는 저자 → build-voice / 둘 다 → 혼합.
5. IP: 프리셋 설계 원본은 마스터에만(배포 제외), 배포본엔 적용된 결과만.

## 선행 조건 / 리스크

- 프리셋 **내용 집필은 대표의 저작 작업** (커리큘럼처럼 사람 영역) — 내가 대신 못 함. 나는 구조·스킬만.
- 정식 1.0 안정화 이후 착수 (지금 벌이면 코어가 흔들림 — 7/9 과속 교훈)
- 프리셋이 개성을 획일화하지 않도록: 항상 "출발점일 뿐, 당신의 문체로 발전"을 원칙에 명시

## 2층 구조 설계 (2026-07-12, 대표 질문 "몇 개가 적정? 축 조합 생성은?"에 대한 Claude 제안)

**1층 — 프리셋 (큐레이션, 5~7종 고정)**: 이름 붙은 완성형 스타일. 5종이 적정 하한, 7종이 상한
(그 이상은 시니어에게 선택 부담 — '폰트 고르기'가 '폰트 공부'가 됨). 대표의 브랜드 자산은 이 층.

**2층 — 문체 맞춤 문답 (조합 생성기)**: 7축(어미·문장리듬·어휘·독자말걸기·사례방식·**전개방식**(대표 제안 추가)·금지)을
저자에게 조합표로 노출하지 않고, **질문 5~7개 → 축 값 결정 → 즉석 예문 2개 생성 → 고르면 확정**하는
피팅룸 방식. 수천 조합이 가능하지만 저자는 질문에 답하고 예문을 고를 뿐이다.

**두 층의 관계**: 프리셋 = 검증된 좌표(품질 보증), 맞춤 문답 = 좌표계 전체(자유도).
권장 UX는 "프리셋에서 시작 → 마음에 안 드는 축 1~2개만 바꾸기"(변형) — 완전 자유 조합은 품질 편차가 크다.

**확장 아이디어 (백로그)**
- 문체 혼합: "따뜻한 멘토 70% + 강연자 30%" 식 가중 결합 (Claude가 잘하는 작업)
- A/B 미리보기: 같은 문단을 두 스타일로 나란히 → 고르게 하기 (선택을 감각의 문제로)
- 닮고 싶은 저자 입력 → 문체 '특징'만 추출해 근접 프리셋+변형 제안 (문장 모방 아님 — 저작권 안전)
- 문체 성장 리포트: build-voice 재분석 때 "프리셋에서 이만큼 당신 것이 되었습니다" (애착 장치)
- 장르팩 확장판: 에세이팩·실용서팩 등 프리셋 추가 묶음 = 판매 가능한 후속 자산

**현재 구현 상태 (07-12 오후)**: **프리셋 7종 전체 정본 확정** (07-12 대표 승인 — 6·7번은 예문 품질 피드백 반영 후 확정. '예문=최고 수준 시연' 원칙 명문화).
**2층 완성**: 문체 좌표계(7축×선택지)를 presets.md에 수록 + tune-voice 스킬(변형/전체 문답, A/B 예문, 축 노출 금지) 신설 — v4.4 페이로드 포함.
남은 백로그: 혼합 가중치·닮고 싶은 저자 매핑·문체 성장 리포트·장르팩.
